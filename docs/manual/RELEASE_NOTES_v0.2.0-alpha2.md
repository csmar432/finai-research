## 🚨 Known issues (carried forward — not blockers)

### 1.ESG financing paper — internal data inconsistencies

The sample `.pdf` / `.docx` at `papers/us_esg_financing/` contains two
incompatible descriptive-statistics tables (one with N=42, one with N=112)
and arithmetic mismatches between prose and tables. This is documented
in detail at `papers/us_esg_financing/AUDIT.md` and
`papers/us_esg_financing/AUDIT_NOTES.md`. **The empirical findings
should not be cited without independent verification.**

Workaround: re-run `scripts/research_framework/pipeline.py --topic
"ESG performance and corporate financing constraints"` against the
canonical yfinance dataset; the regenerated output will override the
stale manuscript.

### 2. `diff-in-diff2` package rename

`scripts/research_framework/modern_did.py` references the PyPI package
`diff-in-diff2`, which **does not exist** on PyPI as of 2026-07-04
(renamed to `csdid` perhaps). Code falls back to a graceful
`EstimatorUnavailableError` if not installed, so functionality is
preserved. Migration plan: update import paths in v0.3.

### 3. `api.star-history.com/svg` endpoint returns 500 for small repos

GitHub's 2026-06-30 restriction on unauthenticated stargazer API
access broke all star-history.com badges for repositories with few
stars (community-wide). README still links to the endpoint — the badge
just renders a 500 fallback. Migration plan: replace with self-hosted
SVG generated by `narayann7/star-history-action` (scheduled for v0.4).

### 4. `count_assets.py` reports `45 research_directions` when registry
           loads, but `12-13` in test docstrings

Two different counting strategies coexist (registry.size vs file count).
Both are recorded in `scripts/count_assets.py` comments. Will be
reconciled when the legacy `_legacy_registry.py` is removed (v0.4).
